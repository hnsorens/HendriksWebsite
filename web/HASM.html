Project Overview

HASM (Hendrik's Assembly Language) is a custom assembly language designed for Linux systems, complete with its own compiler, assembler, and toolchain. Built from scratch in C, this project demonstrates deep understanding of low-level programming, compiler design, and system architecture. HASM provides a simplified yet powerful assembly language that compiles to native x86-64 machine code.

Complete toolchain: Custom compiler, assembler, and linker that generates native ELF executables for Linux.
Low-level mastery: Direct manipulation of machine code, instruction encoding, and binary file formats.

Key Features

Custom Assembly Language

Simplified syntax designed for readability while maintaining full control over hardware resources and memory management.

Complete Compiler

Multi-pass compiler with lexical analysis, parsing, semantic analysis, and code generation phases.

ELF Binary Generation

Generates standard ELF (Executable and Linkable Format) binaries compatible with Linux systems.

x86-64 Support

Full support for x86-64 instruction set architecture with custom instruction encoding and optimization.

Custom Instructions

Extensible instruction set with support for custom instructions and macros for specialized operations.

Optimization Passes

Multiple optimization passes including dead code elimination, constant folding, and register allocation.

Technical Deep Dive

Language Design

  • Simplified syntax inspired by x86 assembly but with improved readability.
  • Support for labels, directives, and custom instruction definitions.
  • Type-safe addressing modes and operand validation.
  • Macro system for code reuse and abstraction.

Compiler Architecture

  • Lexical Analysis: Tokenization and symbol table management.
  • Parsing: Recursive descent parser with error recovery.
  • Semantic Analysis: Type checking and symbol resolution.
  • Code Generation: Instruction selection and binary output.

ELF Format Handling

  • Direct manipulation of ELF file headers and section tables.
  • Proper symbol table generation and relocation handling.
  • Support for different ELF sections (text, data, bss, etc.).
  • Debug information generation for development tools.

Instruction Encoding

  • Manual x86-64 instruction encoding and decoding.
  • Support for various addressing modes and operand types.
  • Optimization of instruction sequences for performance.
  • Custom instruction extension framework.

Optimization Engine

  • Dead code elimination and unreachable code removal.
  • Constant folding and expression simplification.
  • Register allocation and spill optimization.
  • Peephole optimization for instruction sequences.

Project Significance & Learning Outcomes

  • Low-Level Programming: Mastery of x86-64 architecture, instruction encoding, and binary file formats.
  • System Programming: Direct manipulation of ELF files, memory layout, and system call interfaces.
  • Language Design: Principles of creating readable, efficient, and extensible programming languages.
  • Optimization Techniques: Implementation of various compiler optimizations for performance improvement.
  • Toolchain Development: Building complete development tools from assembler to linker.

Technical Challenges Overcome

  • ELF Format Complexity: Understanding and implementing the complex ELF binary format specification.

Real-World Applications

  • System Programming: Foundation for understanding how higher-level languages compile to machine code.
  • Research Platform: Base for experimenting with new language features and optimizations.

How to Build & Run

  1. Ensure you have a C compiler (GCC) and Make installed on your Linux system.
  2. Clone the repository from the HASM folder.
  3. Open a terminal in the HASM directory.
  4. Run the following commands:
make
./hasm input.asm -o output
chmod +x output
./output
